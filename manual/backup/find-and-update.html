# Find And Update

Update 할 Job 조회 및 Job 속성을 일괄로 Update 시킬 수 있다.

  • 화면 구성
    • 상단 조회용 조건 리스트 : Job을 조회 하기 위한 조건이 있는 리스트.
    • 중앙 Update 리스트 : 조회 된 결과에 Update 를 수행 하기 위한 조건과 값이 있는 리스트.
    • 하단 결과 리스트 : Job 조회 / Update 결과 리스트.
  • Update 순서
    1. Master Jobs 화면을 통해 수행하고자 하는 Resource 의 Lock 획득 및 Open.
    2. PCP Toolbar 에 있는 아이콘 을 통해 Job Find And Update 화면 Open.
    3. 조회용 조건 리스트 에서 조건 추가 및 조회 (결과 리스트 에서 조회 내용 확인).
    4. Update 리스트 에서 Update 조건 추가 .
    5. 결과 리스트 에서 Update 하고자 하는 Job (레코드) 선택. (전체선택 , 전체 해제 가능).
    6. Update 리스트 에서 Update 버튼 클릭 으로 Update 수행.
    7. 결과 리스트 의 Action 필드를 통해 Update 결과 확인.
    8. Update 완료 후, Master Job 화면에서 Write/Upload 수행으로 DB 에 반영 한다.(자세한 방법 은 Master Job 화면 참고)

# 조회 조건 리스트

새로운 조건을 추가 , 삭제 할 수 있다. 또 조회 버튼 을 통해 조건에 맞는 Job 을 조회 할 수 있고, Clear 버튼 을 통해 조회 내용을 clear 할 수 있다. 조회 결과는 결과 리스트 에서 확인 할 수 있다. 각 레코드는 AND 조건이다.

  • 컬럼.
    • Action : 수행하고자 하는 Action 정의.
    • Target : 검색 항목. (Target 컨텍스트 메뉴 참고)
    • Operator : 검색 연산자.
    • Value : 검색 값.

# Update 조건 리스트

Update 조건 추가 및 삭제 할 수 있고, 결과 리스트 에 선택된 Job(레코드) 를 Update 한다. 각 Update 조건 레코드는 순차적으로 수행된다.

  • 컬럼.
    • Action : 수행하고자 하는 Action 정의.
    • Target : Update 수행 할 항목. (Target 컨텍스트 메뉴 참고)
    • Operator : Update 연산자. (주요 항목은 Assign, Replace, Clear가 있다)
      • Assign : Job 항목에 Value 를 입력한다.
      • Replace : Job 항목에 있는 String Value 를 Value(replace) 로 치환한다.
      • Clear : Job 항목의 값을 삭제 한다.
    • Value : Operator 에 따라 입력 값 혹은, Update 대상 값.
    • Value(replace) : Replace 값. (Operator 에 따라 사용 유무가 결정된다)

# 결과 리스트

Update 할 Job (레코드) 를 직접 선택, 전체 선택 및 전체 해제 가 가능하다. Update 된 Job(레코드) 를 Rollback 할 수도 있다.

  • 컬럼.
    • Action : Action 결과. (항목으로는 Find, Update, Update Error 이 있다)
      • Find : 조건 검색 완료.
      • Update : Update 완료.
      • Update Error : Update 실패.
    • Server : Server 이름.
    • Resource : Resource 이름.
    • Application : Job Application.
    • Group : Job Group.
    • Name : Job 이름.
    • Type : Job Type.
    • Destription : Job 설명.

# Target 컨텍스트 메뉴

조회 조건 리스트 와 Update 조건 리스트 에 있는 Target 컬럼의 컨텍스트 메뉴이다. 이 메뉴를 통해 Target 을 선택 할 수 있다.

  • Target 컨텍스트 메뉴 항목.
    • General : Job 일반 정보.
    • General(Cyclic) : Job 일반 정보 중 Cyclic 상세 정보.
    • Condition : Job 의 In/Out Condition 정보.
    • Schedule : Job 의 Schedule 정보.
    • Advanced : Job 의 Post Proc 및 Variables 정보.

상세 항목은 Job Properties 의 각 Tab 별 항목과 동일하다.(Cyclic 은 General 에 포함된다.) 자세한 설명은 Job Properties 를 참조하자.

Child 컨텍스트 메뉴
몇몇 Target 항목은 Child Update 조건 이 필요하다. 각 Operator 컬럼에서 Child 컨텍스트 메뉴를 확인 할 수 있다.

  • Child 항목이 있는 Target.
    • Condition -> InCondition
    • Condition -> OutCondition
    • Condition -> Post Porc
    • Advanced -> Post Porc
    • Advanced -> Variables

# 특이사항

# Cyclic 입력값

  • 조회 조건 리스트와 Update 리스트 의 Target 별 Value 포멧
    • Rerun every(unit) : 시간의 단위(M/H) 와 숫자를 함께 입력한다.
      • 예를들어 M15면 15분 주기, H1 이면 1시간 주기를 의미한다.
    • Runat(HHMM) : 수행할 시각(HHMM) 을 콤마(,) 구분으로 여러개 등록 한다.
      • 예를들어 1030,1100,1800 는 10시30분, 11시, 18시 로 등록된다.
