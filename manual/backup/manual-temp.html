# PowerControl-Plus 서버 구성 요소

  • 서버

    스케쥴러 엔진, Client Gateway, Agent Gateway, Database 관리 기능 등을 수행
    2Node Cluster - 원격 배치서버를 2대 까지 연동할 수 있음

  • 리소스

    Job 그룹 - 하나의 Job은 반드시 하나의 리소스를 지정해야한다.
    리소스는 Job을 편집하기 위한 Lock의 단위가 되며, 준비작업 시간을 속성으로 가진다.

  • Job

    스케쥴링 대상이 되는 Batch 단위 업무
    실행 조건이나 Job Type 등 다양한 프로퍼티를 정의할 수 있다.

# 준비작업(New Day Procedure)

  • 리소스 단위로 설정된 시간에 새로 시작할 Job리스트를 DB에 생성하고,
  • (첫 번째 리소스라면)스케쥴링하여 실행할 엔진 쓰레드를 생성한다.

# 스케쥴링 엔진 쓰레드

  • 일자 별로 하나의 엔진 쓰레드가 생성되며,
  • 실시간으로 이벤트를 모니터링 하면서 조건에 부합하는 Job의 실행을 지시하고,
  • 그 결과를 DB에 업데이트 한다.

# today_job_step

/bg-red/today order_id seq type param opcode val and_or
20190911 21 1 IF EXITCODE > 0 AND
20190912 21 2 IF ENDTIME > 1500 AND
20190905 21 3 IF ELASPED > 600 AND
20190905 21 4 THEN NOTOK null null null
20190905 21 5 THEN OK null null null
20190905 21 6 THEN STOPCYCLIC null null null
20190905 21 7 IF TRUE null null null
  • type : IF/THEN
  • param : EXITCODE/ENDTIME/ELASPED
  • opcode: 비교 연산자
  • val: String (Validation Check: Column 타입은 varchar이지만 IF조건을 선택할 경우 숫자(4)만 입력받아야 함)

# Schedule 등록

  • Job의 일정을 등록하는 화면이다.
  • 'Days' or 'Week', 'Months'가 기본이다.
  • Schedule 등록 후 Job 수행여부는 로 알 수 있다.
  • Date에 날짜를 입력할 때는 Calendar영역과 Months영역에 입력이 없어야 한다.
  • Active와 Date를 동시에 입력할 때는 Date가 Active의 기간안에 있어야 한다.

# Calendar Manage

  • 업무 스케줄에 맞게 Calendar를 등록하는 화면이다.
  • Calendar List 등록

    • New -> Create Calendar -> Year(YYYY)해당년 Default, Calendar Name 입력 -> OK -> Calendar List 비활성화, 달력, Week, Preset 설정 -> Apply -> Write -> Upload
      Ex) New -> Create Calendar -> 2020, Bizday -> Write -> Upload

      Write를 활성화하려면 Apply를 클릭하거나, Name의 Description에 입력이 있어야한다.
처음으로 돌릴려면 Search버튼으로 초기화 한다.

      Calendar List를 삭제해도 같은 이름으로 Calendar List를 만들지 못함.

  • Holiday 등록
    • Calendar Manage화면의 Preset에서 -> Edit -> Holiday Manage 팝업 -> New -> Create Holiday ID -> Id, Name, Description 입력 -> 추가된 행에서 오른쪽 마우스 클릭 -> New -> 공휴일을 추가할 때마다 계속 New 해야됨.
