  • 사용자의 정보를 확인하고 변경하는 화면이다.

# 패스워드 변경

  • 패스워드를 변경하는 화면이다.
    • 최소 8자 ~ 최대 16자이내로 입력
    • 반드시 영문, 숫자, 특수문자가 각 1자리이상 포함해야 한다.


계정 생성이나 패스워드 초기화는 admin 사용자만 권한을 가지고 있습니다.
신규 계정의 패스워드는 계정과 동일하며, 최초 로그인시 반드시 패스워드를 변경해야 합니다.
패스워드는 암호화 되어 DB에 저장되므로 관리자도 확인할 수 없습니다.


# 사용자 권한 관리


사용자 등록, 패스워드 초기화, 사용자 권한 관리를 지원한다.

  • PCP에 적용된 사용자 권한 관리는 계정별로 선택된 자원(Resource, Application, Group)에 대해 Action 권한을 부여하는 방식 이다.
  • 메뉴 권한은 Action 카테고리에 포함된다.
  • 화면 구성
    • 상단 User Registration : 사용자 검색 및 추가 /삭제 할 수 있다. 초기 패스워드 및 패스워드 초기화 시, 패스워드는 UserId 과 같다.
    • 좌측 하단 Action Group : Action Group 화면에서 정의한 Group 을 등록 /삭제 할 수 있다.
    • 우측 하단 Auth Resource Group : Auth Group 화면에서 정의한 Group 를 등록 /삭제 할 수 있다.
  • 등록 순서
    1. User Registration 에서 사용자 등록 및 선택 한다.
    2. Action Group 에서 부여하고자 하는 Action Group 추가 /저장 한다.
    3. Auth Resource Group 에서 부여 하고자 하는 Auth Resource Group 을 추가 /저장 한다.

# Action Group

사용자에게 부여할 적절할 Action Group을 정의한다.

  • Action Group은 리소스와 리소스에 대해 허용된 Action의 묶음을 의미한다.
  • Administrator 의 경우 ALL_ACCESS가 지정된 Action Group을 선택한다.
  • 나머지 사용자의 경우에는 의미별로 적절히 나눠진 Action Group 을 선택한다.

# Auth Resource Group

Auth Group 화면 에 정의된 Group Type (RESOURCE, GROUP, APPLICATION) 별 권한를 선택한다.

같은 Group Type 의 항목을 여러개 등록하여 권한을 늘리거나, 다른 Group Type 의 항목을 등록하여 권한 범위를 좁힐 수 도 있다.

# Auth Group

Auth Resource Group 별 Auth Resource 를 부여 할 수 있다. 부여 가능한 Auth Resource 의 Group Type 은 Job properties 항목인 Resource, Group, Application 이 있다.

  • 화면 구성
    • 상단 Auth Resource Group : Auth Resource Group 을 검색 및 추가 /삭제 할 수 있다.
    • 하단 Auth Resource List : Auth Resource Group 의 Group Id 별 상세 Auth Resource 을 등록 /삭제 할 수 있다.
  • 등록 순서
    1. Auth Resource Group 에서 Group Type을 설정 한뒤, 새로운 Group Id 를 추가 및 저장 한다. (Auth Resource Group 참고)
    2. Auth Resource Group 에서 추가된 Group Id 를 선택 한 뒤, Auth Resource List 에서 상세 Auth Resource 을 추가 /저장 한다.

# Auth Resource Group

  • Resource Group (Gruop Type)
    • RESOURCE : Job Properties 의 Resource.
    • GROUP : Job Properties 의 Group.
    • APPLICATION : Job Properties 의 Application.

# Auth Action

메뉴, Today Job Action, Master Job Action 에 대한 실행 권한을 설정하는 화면이다.

  • 화면 구성
    • 상단 Action Group : Action Group 을 검색 및 추가 /삭제 할 수 있다.
    • 하단 Action List : Action Group 의 Group Id 별 상세 Resource Action 을 등록 /삭제 할 수 있다.
  • 등록 순서
    1. Action Group 에서 새로운 Group Id 를 추가 및 저장 한다.
    2. Action Group 에서 추가된 Group Id 를 선택 한 뒤, Action List 에서 상세 Action 을 추가 /저장 한다. 자세한 내용은 Action List 를 참고 한다.

# Action List

  • ALL_ACCESS
    • 사용자에게 모든 Action에 대한 권한을 부여한다.
    • 모든 Action 권한을 가져야 하는 Administrator 인 경우 ALL_ACCESS 권한을 부여한다.
  • ACTION_SELECT
    • 사용자에게 권한을 부여할 Action 항목을 선택한다.
    • MENU ACTION : 사용자에 노출될 메뉴 항목을 지정
    • DIAGRAM JOB ACTION : Today Diagram에서 실행 가능한 Action을 지정
    • MASTER GROUP ACTION : Job 편집화면(Master Jobs)에서 실행 가능한 Action을 지정
    • MASTER ROOT ACTION : Master Jobs에서 Export/Import 에 대한 실행 권한을 지정
