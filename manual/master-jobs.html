# 리소스

  • 배치 작업의 묶음을 리소스라 한다.
  • 일반적으로 배치 서버를 공유하는 비지니스 단위로 할당 하며 Lock을 획득하고 Job에 대해 변경/저장/배포하는 단위가 된다.
  • 준비작업(New Day Procedure)을 수행하는 단위이다. Order Time을 속성으로 가지고 Type에 따라 자동으로 준비작업이 진행될지 수작업으로 진행될지 여부가 결정된다.
  • Job 이름은 중복을 허용하지만 리소스 이름은 서버에서 유일해야 한다.

리소스 조회, 등록, 변경, 삭제 기능을 지원한다. 리소스를 선택하고 을 누르면 Job 편집화면이 열린다.

# 리소스 등록

화면 상단 버튼을 클릭한다.


[step 1. New - 생성]

[step 2. Write - 저장]

[step 3. Lock - 잠금]

[step 4. Uload - 배포]

# 리소스 삭제

  • 삭제할 리소스를 체크하고 Lock을 획득한다.
  • 버튼을 눌러 삭제하고, Wirte -> Upload 순으로 진행한다.
  • 삭제된 리소스는 를 선택하면 조회가 가능하다.
  • 삭제된 리소스는 으로 복구 할 수 있다.

# Job Properties

# General

Job의 기본적인 속성을 조회 하는 화면이다. 화면은 다이어그램에서 Job을 더블 클릭하거나 Job Context 메뉴를 통해 오픈할 수 있다.

  • Schedule 탭은 Master Job 화면을 통해 오픈했을 경우에만 보인다.
항목 설명
Name Job 이름
Confirm Confirm Job 여부 (Confirm Job일 경우 준비 작업때 Hold 상태가 된다)
Cyclic Cyclic Job 여부
Cyclic Option Cyclic Option
Type Job 유형 (Dummy, Command, File Transfer)
Argument Agent 로 전달될 명령어 혹은 실행 조건(File Transfer)
Server 배치서버 이름
Resource Reousrce 그룹 (논리 그룹, 이 단위로 준비작업이 수행된다.)
Owner Agent 명령 실행 계정(Switch User)
Application Application 그룹 (논리 그룹)
Group Group 그롭 (논리 그룹)
Node Node 명, 혹은 hostname
Duration Job 지속 기간 (Duration 참고)
Required Job 이 수행하는데 필요한 Quantity
Schedule-From Job 이 수행될 시작 시각
Schedule-To Job 이 수행될 수 있는 최대 시각 (이 시각 이후 Job 은 pending 상태가 됨)
Description Job 설명

리소스 명은 중복을 허용하지 않지만, Job 이름은 중복을 허용한다.
따라서, 준비작업 후에도 동일한 Job을 Order 할 수 있다. (key=order id)

# Job Type

  • Dummy : 작업이 실행되지는 않으나 Workflow를 시작하거나 필요에 의해서 합칠 때 사용한다.
  • Command : Argument 항목에 정의된 명령을 Agent에 전달하는 일반적인 유형
  • File Transfer : Agent , Batch Server 간 파일 전송하는 유형

# Cyclic Option

그림과 같이 Cyclic 체크박스를 체크 하면 Cyclic Option 영역이 활성화 된다.

  • Rerun Every
    • Rerun Every 에서 분단위(M:MIN) 혹은 시간단위(H:HOUR) 로 주기를 설정 할 수 있다.
    • From Job's 콤보박스에서 S:START, E:END 를 선택 할 수 있고 그 의미는 다음과 같다.
      • Start : 다음 수행 주기를 Job 시작 시각부터 계산한다.
      • End : 다음 수행 주기를 Job 종료 시각 부터 계산한다.
    • MaxRun 에서 최대 반복 횟수를 정할 수 있다. (범위: 1~99, 0=Max)
  • Runat
    • 버튼을 통해 수행할 시각 추가, 버튼을 통해 삭제 한다.

# Duration

  • 완료되지 않은 Job은 Duration(Day)이 지나면 Expire 처리를 한다. (범위: 1~99, 0=Max)
  • Hold, Error, Running, Unknown 상태인 Job은 Expire 대상에서 제외됨

# Condition

Job이 정상적으로 완료되면 Condition out 이 발행되며 Conditon은 Workflow를 구성할 때 경로를 결정하는 중요한 파라미터가 된다.

다른 Job의 Out-Condition을 In-Condition으로 등록하면 두 Job은 서로 선후행 관계가 성립 됨

컨디션은 다음의 성격을 가진다.

  • 한 개 이상 Condition 등록이 가능하다.
  • Condition in 은 And 혹은 Or 조건을 선택할 수 있다.
  • Condition out 은 완료 후 컨디션을 발행(Add) 혹은 발행된 컨디션을 삭제(Del)할지 선택 가능하다.
  • Condition은 선택적으로 등록하지 않거나 다른 Job과 중복해서 등록할 수 있다.
  • API나 Condition 관리 도구(화면)를 통해서도 Condition 발행이 가능하다.

# Condition 등록

  • 서버에 등록된 컨디션 중에 선택 하거나 신규 혹은 복사를 한다.
  • Effect: Conditionout 은 발행(Add) 뿐아니라 이미 발행된 Condition을 삭제(Del) 할 수 있다.
  • Conditionin 의 경우 등록 순서에 따라 And/Or 조건이 적용되며, 마지막 레코드의 조건은 무시된다.

# Schedule

Schedule 탭에서 Job의 실행(Order)조건을 입력하고, Forecast를 통해 확인할 수 있습니다.
PCP는 영역별로 입력된 값을 조합하여 스케쥴링 여부를 결정합니다.

# 실행(Order) 조건

  1. 캘린더를 입력하는 경우
  • Days 와 Week에 선택된 값이 있다면 콤보박스의 조건(And/Or)으로 실행 여부를 판단한다.
  • 선택된 캘린더가 존재하면 캘린더의 실행 조건과 함께 판단한다.(And 연산)
  • 마지막으로 Months 와 Active 조건을 더하여 실행 여부를 결정한다.
  • Days 혹은 Week 중 어느 한 쪽만 선택된 값이 없다면 선택되지 않은 항목은 전체 선택과 동일한 조건으로 간주한다.
  1. 일자를 지정하는 경우
  • Date 항목을 입력했을 경우에는 Active 조건만 추가할 수 있다.
  • 다른 항목이 입력되었다면 저장 시점에 화면에서 오류로 처리한다.

캘린더, Days, Week 가 모두 빈칸이면 해당 Job은 실행되지 않습니다.
Active 항목이 빈칸이면 전체 기간을 의미 합니다.

# Forecast

Forecast를 통해 Schedule 탭에서 입력한 조건의 결과를 캘린더 형태로 확인할 수 있다.

# Advanced

그 밖에 Job 완료 후에 후속 작업을 수행하거나, Schedule From/To 항목을 대체할 변수를 지정할 수 있다.

# Post Proc

Post Proc 영역의 버튼을 클릭하면 테이블에 새로운 레코드가 생성된다. IF절과 THEN 모두 하나 이상의 레코드 등록이 가능하다. 등록 후 Use PostProc를 체크 해야 Runtime 중에 Post Proc이 수행된다.

  • 입력 가능한 값 및 설명
IF/THEN Param Operator/Level 설명
IF EXITCODE ==, >, <, >=, <= Job Exitcode
IF ENDTIME ==, >, <, >=, <= Job 종료 시각 (hhmmss)
IF ELAPSED ==, >, <, >=, <= Job 수행 시간 (sec)
IF TRUE N/A True
THEN OK N/A Job 상태변경 Complete
THEN NOTOK N/A Job 상태변경 NotOk
THEN STOPCYCLIC N/A Cyclic Job 일 경우, Stop 시킴
THEN ALARM WINDOW WARN, INFO, CRITICAL 알람창(Popup) 으로 메세지 전송
THEN EVENT MESSAGE WARN, INFO, CRITICAL Event log 에 message 출력

메세지 컬럼에는 시스템 변수를 사용할 수 있습니다. ex) "%JOBNM% is Error!"
사용 가능한 변수는 Tool>Valiables 에서 확인 가능 합니다.

# Variables

준비작업 시, 등록된 변수 정보를 획득하여 Schedule From/To 항목을 대체한다.
등록된 변수는 Use Variables 를 체크 해야 적용이 된다.

Name: From 혹은 To
Var: 사용자 등록 변수 명

  • Var에 입력 가능한 변수명은 Tools > Variables 에서 사용자가 직접 등록한다.
  • Job 실행 시간을 변수로 지정하면 특정 기간의 시간 조건을 Job을 재 등록하지 않고 변수의 값만 변경하여 적용할 수 있다.
  • 사용자 변수의 기본 값과 변경 적용 일자등은 Variables 관리 화면에서 할수 있다.

# Master Job Editor

DB로 부터 Job 정보를 Load 하여 Flowchart 형태로 확인하고, Job 변경관리를 도와주는 도구이다.
Today Order, Copy To/Move To 등의 부가기능도 제공한다.

# Job 신규 등록

Editor는 리소스 관리 화면에서 를 클릭하거나, Import 를 하면 열린다.

  1. 리소스 Lock 획득
  • Resource 영역 컨텍스트 메뉴 -> Resource -> Lock 클릭
  • Lock 을 획득하면 다음과 같이 Resource 이름 오른쪽에 Lock 정보를 확인할 수 있다.
  1. Job 추가
  • Resource 영역 컨텍스트 메뉴 -> Add Job 클릭으로 Job Properties 화면 Open
  • Job Properties 화면에서 Job 상세 정보 입력 (Job Properties 화면 참고)
  • Job 등록을 하면 다음과 같이 새로운 다이어그램이 생성된다.
  • Job 복사기능으로도 새로운 Job 을 생성 할 수 있다. Job 다이어그램 컨텍스트 메뉴 -> Copy(&Edit) 혹은 Copy
  1. Write 수행
  • Resource 영역 컨텍스트 메뉴 -> Resource -> Write 클릭
  • Write 가 수행되면 다음과 같이 다이그램 상태가 바뀐다.
  1. Upload 수행
  • Resource 영역 컨텍스트 메뉴 -> Resource -> Upload 클릭

# Job 삭제

삭제 전 Resource Lock 및 삭제 후 Write/Update 는 Job 등록과 동일하다.

  1. Resoruce Lock 획득
  2. 다이어그램 에서 삭제 할 Job 선택 및 Delete 수행
  • Job 다이어그램 컨텍스트 메뉴 -> Delete 클릭
  1. Wrtie 수행
  2. Upload 수행

# Job 복구(Undelete)

이미 삭제된 Job 에 대해서 복구가 가능하다. 복구 전 Resource Lock 및 복구 후 Write/Update 는 Job 등록과 동일하다.

  1. Resoruce Lock 획득
  2. Undelete list 화면 Open
    • Resource 영역 컨텍스트 메뉴 -> Undelete 클릭
  3. Undelete 화면 에서 Undelete 수행
  4. Wrtie 수행
  5. Upload 수행

# Copy To

연결된 Job의 일부 그룹을 다른 Job의 후행으로 붙여넣기를 할 수 있다.

  • Copy To 절차
  1. Shift 를 누르고 복사하고자 하는 그룹의 Root Job을 Select 상태로 만든다.
  2. 붙여넣기를 하고자하는 Job 위에 마우스를 올리고 우클릭 -> Copy To를 선택한다.
  3. 복사후 생성될 Job 이름의 prefix를 입력한다.
  4. OK버튼 클릭
  • Copy To 결과

# Move To

연결된 Job의 일부 그룹을 다른 Job의 후행으로 이동시킬 수 있다.

  • Move To 절차
  1. Shift 를 누르고 이동하고자 하는 그룹의 Root Job을 Select 상태로 만든다.
  2. 붙여넣기를 하고자하는 Job 위에 마우스를 올리고 우클릭 -> Move To를 선택한다.
  • Move To 결과

# Job Order

개별 Job을 동적으로 준비작업 수행 할 수 있다.

  • Job Order 절차
  1. Order Job 선택
  2. 선택 메뉴에서 Order Date를 확인하고, 옵션을 선택한다.
  3. OK버튼 클릭
  • Job Order 결과
  • Order Option
    • OrderDate
      • ODAT (시스템일자) 혹은 사용자 선택
    • Upload Job and Ordering : 변경(write) 된 Job 인 경우, Upload 수행 후 Order를 수행한다.
    • Force Order(Ignore Schedule data) : 스케쥴링 조건에 맞지 않더라도 강제로 Order 한다.
    • Order With Hold : Hold 상태로 Order 한다.

# 컨텍스트 메뉴

# Resource 영역 켄텍스트 메뉴

  • 컨텍스트 메뉴 설명
    • Add Job : 새로운 Job 추가 (Job preperties 화면 Open)
    • View
      • Write List : Write 대상 출력 화면 Open
      • Upload List : Upload 대상 출력 화면 Open
      • All List : Write List, Update List 출력 화면 Open
    • Resource
      • Lock : Resource Lock 획득
      • Unlock : Resource Lock 반납
      • Download : 마지막으로 Upload (배포) 했던 내용 Download (Overwrite)
      • Write : 작업중인 Job 정보 DB 저장 (배포 아님)
      • Reload : Resource 다시 Load
      • Upload : DB 에 저장된 내용 배포
    • Undelete : Deleted Job 리스트 화면 Open (삭제했던 Job 복구 가능)
    • Export : 등록된 Resource, Job 정보를 json 형태의 파일로 출력
    • Import : Json 형태의 Resource, Job 정보를 화면으로 Load
    • Reset Master Editor : Master Jobs 화면 Open
    • Collapse : 확장 되어 있는 Resource 영역을 축소 함
    • Expend: 축소 되어 있는 Resource 영역을 확장 함
    • Job Order All : 해당 Resource 에 등록되어 있는 모든 Job에 대해 Job Order 수행

# Job 다이어그램 컨텍스트 메뉴

  • 컨텍스트 메뉴 설명
    • Copy(&Edit) : 선택된 Job 복사 (새로운 Job 추가) 후 Job preperties 화면 Open
    • Copy : 선택된 Job 복사 (새로운 Job 추가)
    • Delete :선택된 Job 삭제
    • Clear
      • Prerelation : 선행 Job Relation 삭제 (In Condition 삭제)
      • Postrelation : 후행 Job Relation 삭제 (Out Condition 삭제)
    • Order Job : 선택된 Job 을 특정 날짜로 준비작업 즉시 수행
    • Properties : Job Preperties 화면 Open

# 데이타 저장 및 배포관리

  • 중요 데이타인 리소스, 캘린더, Job 정보는 변경 후에 write와 upload를 완료해야 서버에 반영됨
    • wirte : 화면에서 변경한 데이터를 DB에 저장한다.
    • upload : 일종의 배포 과정으로 DB에 마지막으로 저장된 데이터를 서버 적용 대상 버전으로 변경
    • download : 마지막으로 저장된 레크드를 upload 상태의 데이터로 대체
